# Migrations

OpenAD Zero uses Alembic for schema lifecycle.

## Apply

```bash
make migrate
```

This runs `alembic upgrade head` inside the API container. For host-local development, run `make migrate-local` or `scripts/migrate.sh`, which executes `alembic upgrade head` from `backend/`.

## Create

```bash
./scripts/migration-new.sh "add table"
# or
make migration-new MSG="add table"
```

Review autogenerated migrations before committing.

## Rollback

```bash
cd backend
alembic downgrade -1
```

## Dev reset

```bash
make db-reset-dev
```

The reset script asks for `RESET-OPENADZERO-DEV` before deleting the dev Docker Postgres volume.
