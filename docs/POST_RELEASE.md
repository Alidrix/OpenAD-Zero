# Post-release Checklist

After publishing `v0.1.0-rc1`:

## GitHub release

- [ ] Verify release page exists.
- [ ] Verify pre-release flag is enabled.
- [ ] Verify release notes render correctly.
- [ ] Verify source archives are available.

## Repository

- [ ] Verify README links.
- [ ] Verify SECURITY.md.
- [ ] Verify CodeQL workflow starts.
- [ ] Verify Dependabot is visible in Insights / Dependency graph.
- [ ] Verify CI is green on tag or main.

## Issues and milestones

- [ ] Create milestone `v0.1.0`.
- [ ] Create milestone `v0.2.0`.
- [ ] Convert backlog items into GitHub issues if desired.
- [ ] Label known limitations.

## Communication

- [ ] Mention that this is a release candidate.
- [ ] Mention safe-by-default boundaries.
- [ ] Mention known limitations.
- [ ] Mention that BloodHound CE is optional.
